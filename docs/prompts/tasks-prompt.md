Utwórz lub zaktualizuj `06-tasks.md` na podstawie `04-mvp.md` i `05-tech-spec.md`.

Reguły:
- Jeśli `04-mvp.md` nie istnieje, zatrzymaj się i napisz tylko: `Brak 04-mvp.md. Nie można przygotować 06-tasks.md.`
- Jeśli `05-tech-spec.md` nie istnieje, zatrzymaj się i napisz tylko: `Brak 05-tech-spec.md. Nie można przygotować 06-tasks.md.`
- Pisz zwięźle i konkretnie.
- Nie dodawaj nowych funkcji spoza MVP.
- Nie zapisuj luźnych pomysłów.
- Nie twórz epików udających taski.

Każdy task musi zawierać:
- identyfikator,
- krótki tytuł,
- typ: `new`, `refactor`, `test`, `docs`, `infra`,
- cel,
- zakres,
- wynik końcowy,
- zależności, jeśli istnieją.

Task musi być mały i możliwy do wykonania.
Jeśli zadanie jest zbyt szerokie, podziel je na mniejsze.

Jeśli funkcjonalność rozwija istniejący kod, uwzględnij:
- refaktor,
- integrację,
- migracje,
- porządkowanie obecnego rozwiązania,
jeśli są konieczne do MVP.
